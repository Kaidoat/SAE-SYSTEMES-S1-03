-- Utilisation de la Machine Virtuelle --

Consignes de la SAÉ

Pour la partie système, nous avions plusieurs consignes à executer dans la machine virtuelle. Cet onglet explique toutes ces consignes.

Ajout de comptes avec la commande adduser

Pour ajouter des comptes, il faut utiliser la commande sudo adduser comme suit :

sudo adduser nom_utilisateur

La commande --allow-bad-names permet d'accepter des caractères normalement interdits dans les noms d'utilisateur. Sinon, vous pouvez également modifier la configuration de adduser.conf avec la commande suivante :

sudo nano /etc/adduser.conf

Attribution des droits d'administrateur

Pour que le compte admin_Gr06_Trinome05 ait les droits d'administrateur, il faut l'ajouter au groupe des administrateurs (ou au groupe sudo) avec la commande suivante :

sudo usermod -aG sudo admin_Gr06_Trinome05

Une fois cette commande exécutée, le compte admin_Gr06_Trinome05 aura les privilèges administratifs nécessaires.

Personnalisation de l'invite de commande

Il est possible de personnaliser l'invite de commande (prompt) en utilisant des variables. Par exemple, voici quelques options :

  • \t : affiche l'heure actuelle
  • \u : affiche le nom de l'utilisateur
  • \H : affiche le nom de l'hôte
  • \W : affiche le répertoire courant

Si vous voulez que l'invite de commande affiche seulement le nom de l'utilisateur, vous pouvez modifier PS1 comme suit :

PS1="\\u "

Cependant, cette modification ne persistera que tant que vous ne changez pas de compte. Pour que ce changement soit permanent, même après une déconnexion, vous devez modifier le fichier .bashrc comme suit :

nano ~/.bashrc

Ajoutez ou modifiez la ligne correspondante pour appliquer la personnalisation de manière persistante.
